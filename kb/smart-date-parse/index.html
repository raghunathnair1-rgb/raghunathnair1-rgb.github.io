Smart date/time parsing & due field

Build a natural-language date/time resolver that maps relative terms (today, tomorrow, this/next <weekday>, next week/month, in N days, end of month), explicit formats (27 Jan, 27/1, 10/5/2024), and time phrases (at 3pm, 10:30, noon, midnight) to a concrete due datetime, storing an optional date, optional time (all-day when absent), optional duration/end-time, the raw NL string, and a timezone mode (fixed named zone vs floating/local-day), treating date-only input as all-day and always resolving to the next future occurrence.

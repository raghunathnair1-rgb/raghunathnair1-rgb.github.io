Offline-first sync engine & no-backend MVP

Implement a local-first store (IndexedDB, localStorage fallback) as the read source of truth plus a monotonic sync_token, routing every write through a durable command queue of {type, uuid, temp_id, args} objects that POST in batches to /sync, applying the returned incremental delta and reconciling temp_id→real_id; maintain a WebSocket (long-poll fallback) delivering 'sync-needed' pings that trigger incremental pulls, resolve concurrent edits last-write-wins per field, and keep the repository layer's method shapes mirroring the Sync command API so a client-only MVP can later swap in the real backend untouched.
